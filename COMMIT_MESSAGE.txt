Add database listing functionality for security auditing

- Add ListDatabases method to DatabaseSchemaExtractor class
  - Lists all databases accessible with a given connection string
  - Connects to master database to query sys.databases
  - Returns comprehensive security information for each database
  - Includes permission checks: CanConnect, CanViewDefinition, CanCreateTable
  - Provides database metadata: owner, state, recovery model, collation, etc.

- Add DatabaseListResult and DatabaseInfo data models
  - DatabaseListResult: Contains server info, authentication type, database count
  - DatabaseInfo: Per-database details with security-relevant permissions
  - Includes error handling and detailed error information

- Add list-databases API endpoint
  - POST /api/files/list-databases
  - Accepts connection string and optional timeout
  - Returns complete database list with security information
  - Useful for security auditing and access reviews

- Add list_databases MCP tool
  - Exposes database listing via Model Context Protocol
  - Supports all SQL Server connection types (local, LocalDB, Azure SQL)
  - Includes examples in EXAMPLES.md

- Update all documentation
  - DatabaseExtractor README: Added database listing section with security use cases
  - FileDownloadApi README: Added list-databases endpoint documentation
  - McpServer EXAMPLES.md: Added list_databases tool examples

This feature enables security teams to:
- Audit database access permissions
- Identify databases accessible with given credentials
- Check for over-privileged accounts
- Verify principle of least privilege compliance
- Perform security access reviews
