feat: Add database restore functionality via API and class library

Add complete database restore capability to restore DACPAC/BACPAC files to
target databases with support for both local SQL Server and Azure SQL Database.

Changes:
- Add RestoreDatabase() method to DatabaseSchemaExtractor class
  * Supports both DACPAC (schema only) and BACPAC (schema + data) files
  * Automatically detects file type by extension
  * Creates new databases or upgrades existing ones
  * Includes comprehensive error handling and progress reporting
- Add POST /api/files/{filename}/restore endpoint to FileDownloadApi
  * Accepts JSON request with target connection string and database name
  * Returns detailed success/error responses
  * Validates input parameters and file existence
- Update file listing to include both .dacpac and .bacpac files
- Add project reference from FileDownloadApi to DatabaseExtractor
- Update API documentation with restore endpoint details and examples

Technical implementation:
- RestoreDatabase() uses DacServices.ImportBacpac() for BACPAC files
- RestoreDatabase() uses DacServices.Deploy() for DACPAC files
- Supports upgradeExisting flag to control database creation/upgrade behavior
- Proper error handling for file not found, invalid parameters, and DacFx errors

API Endpoint:
POST /api/files/{filename}/restore
Request: { targetConnectionString, targetDatabaseName, upgradeExisting }
Response: { success, message, filename, targetDatabaseName }

Usage example:
POST http://localhost:8080/api/files/MyDatabase.bacpac/restore
{
  "targetConnectionString": "Server=(local);Database=RestoredDB;Integrated Security=True;",
  "targetDatabaseName": "RestoredDB",
  "upgradeExisting": false
}
